# Database

PostgreSQL 17 + pgvector setup, schema design, and migration notes.

---

## Why This Stack

- **PostgreSQL** — industry standard, supported everywhere, handles relational data well
- **pgvector** — adds vector similarity search to Postgres so we can do RAG without a separate vector database
- **Railway pgvector-pg17 template** — PostgreSQL 17 with pgvector pre-installed, one-click deploy

We rejected Supabase (extra platform) and Railway plain Postgres (no pgvector) in favour of this template.

## Schema

### users

Mirrors Clerk users. Created via Clerk webhook when someone signs up.

| Column        | Type            | Notes                |
| ------------- | --------------- | -------------------- |
| id            | String (PK)     | UUID generated by us |
| clerk_user_id | String (unique) | Clerk's user ID      |
| email         | String (unique) |                      |
| created_at    | DateTime        |                      |

### bots

Each HR team's AI assistant.

| Column          | Type                | Notes                            |
| --------------- | ------------------- | -------------------------------- |
| id              | String (PK)         | UUID                             |
| user_id         | String (FK → users) | Owner                            |
| name            | String              | e.g. "HR Assistant"              |
| system_prompt   | Text                | How the bot should behave        |
| welcome_message | String              | First message shown to new hires |
| is_active       | Boolean             | Default true                     |
| created_at      | DateTime            |                                  |

### documents

Uploaded files per bot.

| Column     | Type               | Notes                                 |
| ---------- | ------------------ | ------------------------------------- |
| id         | String (PK)        | UUID                                  |
| bot_id     | String (FK → bots) |                                       |
| filename   | String             | Original filename                     |
| file_url   | String             | R2 storage key                        |
| status     | String             | uploaded → parsing → indexing → ready |
| created_at | DateTime           |                                       |

### document_chunks

The core of RAG — parsed text + vector embeddings.

| Column      | Type                    | Notes                                         |
| ----------- | ----------------------- | --------------------------------------------- |
| id          | String (PK)             | UUID                                          |
| document_id | String (FK → documents) |                                               |
| content     | Text                    | The actual text chunk                         |
| chunk_index | Integer                 | Position in the original document             |
| embedding   | Vector(1536)            | OpenAI embedding — used for similarity search |
| created_at  | DateTime                |                                               |

### conversations

One conversation = one new hire's chat session.

| Column     | Type               | Notes                            |
| ---------- | ------------------ | -------------------------------- |
| id         | String (PK)        | UUID                             |
| bot_id     | String (FK → bots) |                                  |
| session_id | String             | Identifies a unique chat session |
| created_at | DateTime           |                                  |

### messages

Individual messages within a conversation.

| Column          | Type                        | Notes                 |
| --------------- | --------------------------- | --------------------- |
| id              | String (PK)                 | UUID                  |
| conversation_id | String (FK → conversations) |                       |
| role            | String                      | "user" or "assistant" |
| content         | Text                        | The message text      |
| created_at      | DateTime                    |                       |

## Connection URLs

Railway gives you two URLs. Use the right one in the right place:

| URL                            | When to use                                          |
| ------------------------------ | ---------------------------------------------------- |
| `DATABASE_URL` (internal)      | Inside Railway — backend talking to DB in production |
| `DATABASE_PUBLIC_URL` (public) | Local development from your Mac                      |

Always replace `postgresql://` with `postgresql+asyncpg://` for SQLAlchemy async.

```bash
# Local .env — use the PUBLIC url
DATABASE_URL=postgresql+asyncpg://postgres:pass@yamabiko.proxy.rlwy.net:PORT/railway
```

## Migrations (Alembic)

### Run all pending migrations

```bash
alembic upgrade head
```

### Create a new migration after changing models

```bash
alembic revision --autogenerate -m "describe your change"
alembic upgrade head
```

### Roll back one migration

```bash
alembic downgrade -1
```

## Migration Gotchas

1. **Missing greenlet on Python 3.9**

   ```bash
   pip install greenlet
   ```

2. **pgvector import missing in auto-generated migration**
   Add this at the top of the migration file:

   ```python
   import pgvector.sqlalchemy
   ```

3. **vector type not found**
   Add this as the first line of the `upgrade()` function:

   ```python
   op.execute('CREATE EXTENSION IF NOT EXISTS vector')
   ```

4. **SSL rejection from pgvector template**
   The pgvector Railway template does NOT use SSL. Remove `connect_args={"ssl": "require"}` from both `db/session.py` and `alembic/env.py`.

## Verifying pgvector is Installed

```bash
python3 -c "
import asyncio, os, asyncpg
from dotenv import load_dotenv
load_dotenv()

async def test():
    url = os.getenv('DATABASE_URL').replace('postgresql+asyncpg://', 'postgresql://')
    conn = await asyncpg.connect(url)
    result = await conn.fetchval('SELECT extversion FROM pg_extension WHERE extname = \$\$vector\$\$')
    print(f'pgvector version: {result}')
    await conn.close()

asyncio.run(test())
"
```

Expected output: `pgvector version: 0.8.2`
